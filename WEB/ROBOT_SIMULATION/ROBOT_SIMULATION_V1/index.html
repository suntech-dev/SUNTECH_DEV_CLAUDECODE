SunTech Co-Robot Sewing Simulation 사용 메뉴얼

1. 프로그램 소개

이 프로그램은 협동로봇(Co-Robot)과 컴퓨터재봉기가 함께 작업하는 공정을 시뮬레이션합니다.

대상 제품: SunTech ST-ES612, ST-EA612 Robot 모델 기준으로 제작했습니다.

2. 시작하기 - 모드 선택

프로그램을 시작하면 재봉기 수량을 선택하는 화면이 나타납니다.

모드 설명 배치
2대 모드 재봉기 2대 운용 좌측(M1) + 우측(M2)
3대 모드 재봉기 3대 운용 좌측(M1) + 상단(M2) + 우측(M3)

Tip: 상단 헤더의 [모드 변경] 버튼으로 언제든 다시 선택할 수 있습니다.

3. 기본 설정

항목 설명 기본값
작업인원 동시에 작업하는 인원 수. PPH(시간당 생산성) 계산에 사용됩니다. 2명
Pallet 준비시간 작업자가 자재 테이블에 새 원단(Pallet)을 준비하는 데 걸리는 시간 58초
Pallet 수량 시스템에서 순환하는 총 Pallet 개수 (3~7개). 재봉기 수량보다 많으면 자재 준비를 미리 할 수 있어 가동률이 높아집니다. 5~6개
Pallet 수량:
• Pallet 수량이 많을수록 자재 준비 대기시간이 줄어들어 생산성 향상

4. 작업 시간 설정

4-1. 로봇 속도 선택

로봇의 동작 속도를 프리셋 또는 직접입력으로 설정할 수 있습니다.

모드 설명
직접입력 사용자가 각 재봉기의 시간값을 자유롭게 입력합니다 (기본값)
75% ~ 100% 로봇 속도 프리셋을 선택하면 초기 Load, Unload & Load, 복귀 시간이 자동으로 설정됩니다
참고: 재봉 시간은 로봇 속도와 무관하므로 항상 직접 수정 가능합니다.

속도별 기준값 (75% / 85% / 100%):

재봉기 항목 75% 85% 100%
재봉기 1 초기 Load 12.5초 11초 8.5초
Unload & Load 17.5초 15.5초 13초
복귀 10.5초 9초 7.5초
재봉기 2 초기 Load 13.5초 11.8초 9초

Tip: 80%, 90%, 95% 등 중간값은 75-85, 85-100 구간별 선형 보간으로 자동 계산됩니다.

4-2. 시간 항목 설명

각 재봉기별로 4가지 시간을 설정합니다 (초 단위):

항목 설명
초기 Load 시간 로봇이 처음으로 Pallet를 재봉기에 Load 하는 시간
Unload & Load 시간 재봉 완료한 Pallet Unload + 새 Pallet load를 360도 회전 기능으로 연속 수행 하는 시간 (2번째부터)
복귀(회수) 시간 로봇이 재봉 완료한 Pallet를 회수 후 로봇 원점 위치로 돌아오는 시간
재봉 시간 재봉기가 봉제 작업을 완료하는 데 걸리는 시간
참고: 재봉 시간이 길수록 로봇이 다른 재봉기를 서비스할 여유가 생깁니다. 반대로 재봉 시간이 짧으면 로봇이 바빠져 재봉기 대기시간이 늘어납니다.

5. 시뮬레이션 시간 설정

시뮬레이션을 실행할 총 시간을 설정합니다.

  • 최소 1시간 ~ 최대 24시간까지 설정 가능
  • 시간과 분을 각각 선택할 수 있습니다
  • 24시간 선택 시 분은 00분만 가능합니다

6. 시뮬레이션 실행

두 가지 실행 방법이 있습니다:

빠른계산

  • 결과를 즉시 계산합니다
  • 애니메이션 없이 빠르게 결과만 확인
  • 빠르게 결과값을 알고 싶을 때 사용

실시간계산

  • 로봇의 움직임을 시각적으로 확인
  • 속도 조절 가능 (0.1x ~ 100x 로그 스케일)
  • 타임라인 슬라이더로 되감기/건너뛰기
  • Pallet 운반 상태 시각화
  • 일시정지 / 다시시작 가능
  • 작동 과정을 이해하고 싶을 때 사용

실시간계산 모드 상세

기능 설명
실시간 통계 바 시뮬레이션 진행 중 각 재봉기의 생산량, 대기시간, 로봇 가동/대기 시간이 실시간으로 표시됩니다
로봇 이동 애니메이션 로봇이 테이블 ↔ 재봉기 사이를 이동하는 경로와 경과시간이 시각적으로 표시됩니다
재봉기 진행률 바 각 재봉기의 봉제 진행 상황이 프로그레스 바로 표시됩니다
상태 표시 로봇과 재봉기의 현재 상태(봉제중, 대기중, 이동중 등)가 텍스트로 표시됩니다
속도 미세 조절 0.1x ~ 100x 범위의 로그 스케일 슬라이더로 재생 속도를 세밀하게 조절할 수 있습니다. 프리셋 버튼(0.1x, 0.5x, 1x, 5x, 10x, 50x, 100x)도 제공됩니다
Pallet 운반 시각화 로봇이 이동할 때 운반 상태가 표시됩니다. 새 Pallet 운반 시 파란색 P 뱃지, 완성품 운반 시 금색 ✓ 뱃지가 로봇 아이콘에 나타납니다
타임라인 슬라이더 시뮬레이션 진행 상황을 슬라이더로 표시하며, 드래그하여 특정 시점으로 이동할 수 있습니다. -10초/+10초 버튼으로 빠르게 앞뒤로 이동할 수 있습니다

이벤트 로그

시뮬레이션 실행 시 모든 이벤트가 테이블 형태로 기록됩니다.

  • 시간(초): 이벤트 발생 시점
  • 이벤트 유형: 출발, 로딩, 봉제완료, 생산 등
  • 재봉기: 해당 이벤트의 대상 재봉기
  • [접기/펼치기] 버튼으로 긴 로그를 축약하거나 펼칠 수 있습니다

7. 결과 해석

7-1. 재봉기별 결과

항목 의미
총 생산수량 각 재봉기에서 완성된 제품의 개수
총 대기시간 재봉 완료 후 로봇이 올 때까지 기다린 시간 (짧을수록 효율적)

7-2. 종합 지표

항목 의미
총 생산수량 모든 재봉기에서 완성된 제품의 합계
로봇 가동률 전체 시간 중 로봇이 실제로 일한 비율 (높을수록 로봇 활용도 높음)
로봇 총 가동시간 로봇이 이동, 로딩, 언로딩 등 실제 작업한 시간의 합계
시뮬레이션 시간 설정한 총 시뮬레이션 시간
PPH Person Per Hour - 작업자 1인이 1시간에 생산하는 수량
계산식: 총 생산수량 ÷ 시간(h) ÷ 작업인원
재봉기 총 가동률 전체 재봉기의 평균 봉제 가동 비율 (높을수록 재봉기 유휴시간 적음)
재봉기 총 가동시간 모든 재봉기의 실제 봉제 작업 시간 합계

7-3. 효율 개선 방안

시뮬레이션 결과에서 대기시간이 많거나 가동률이 낮다면 다음 방안을 검토하세요:

개선 방안 효과 적용 상황
Pallet 준비시간 단축 Unload-only 발생 감소, 재봉기 대기시간 단축 준비시간이 재봉시간에 근접할 때
Pallet 수량 증가 자재 준비 대기 감소, 연속 작업 가능 로봇이 자주 "Pallet 대기" 상태일 때
로봇 속도 증가 재봉기 서비스 간격 단축 로봇 가동률이 낮고 재봉기 대기시간이 길 때
💡 Unload-only 현상 이해하기

Unload-only란? 로봇이 완료품을 회수할 때 새 Pallet이 준비되지 않아, 완료품만 회수하고 재봉기를 EMPTY 상태로 두는 현상입니다.

발생 원인: Pallet 준비시간(예: 58초)이 재봉시간(예: 63초)에 근접하면, 로봇이 완료품을 테이블에 가져와도 새 자재가 아직 준비되지 않습니다.

영향: Unload-only가 많이 발생하면 재봉기가 EMPTY 상태에서 대기하는 시간이 늘어나 총 대기시간이 급증합니다.

해결책:
  • Pallet 준비시간을 줄여 자재 공급 속도 향상
  • Pallet 수량을 늘려 여유 자재 확보
  • [차선책]작업인원 증가로 Pallet 준비 병렬화

8. 결과 차트

시뮬레이션 실행 후 결과 영역 하단에 4가지 차트가 자동으로 표시됩니다.

차트 유형 설명
재봉기별 생산량 막대 차트 각 재봉기의 완성 수량을 비교합니다. 재봉기 간 생산 편차를 한눈에 파악할 수 있습니다.
재봉기별 대기시간 막대 차트 각 재봉기가 로봇을 기다린 총 시간을 비교합니다. 병목 구간을 확인할 수 있습니다.
로봇 시간 활용 도넛 차트 로봇의 가동시간 vs 대기시간 비율을 표시합니다. 중앙에 가동률(%)이 표시됩니다.
재봉기 시간 활용 도넛 차트 전체 재봉기의 봉제시간 vs 유휴시간 비율을 표시합니다. 중앙에 가동률(%)이 표시됩니다.
활용 Tip: 막대 차트에서 특정 재봉기의 대기시간이 유독 길다면, 해당 재봉기의 재봉 시간이 짧거나 로봇 이동 경로상 우선순위가 낮은 것일 수 있습니다.

9. 시나리오 저장 및 비교

다양한 설정으로 시뮬레이션을 실행한 뒤 결과를 저장하고 비교할 수 있습니다.

9-1. 시나리오 저장

항목 설명
저장 방법 시뮬레이션 실행 후 결과 영역의 [시나리오 저장] 버튼을 클릭합니다
시나리오 이름 저장 시 이름을 입력할 수 있습니다 (예: "속도85% 3대", "팔레트7개 테스트")
최대 저장 수 최대 5개까지 저장 가능합니다
저장 내용 기본 설정, 입력 시간값, 시뮬레이션 결과가 모두 함께 저장됩니다

9-2. 시나리오 비교

2개 이상의 시나리오가 저장되면 자동으로 비교 섹션이 표시됩니다.

비교 항목 설명
성능 비교표 총 생산량, PPH, 가동률 등 주요 지표를 테이블로 나란히 비교합니다. 최고값은 녹색 하이라이트로 표시됩니다.
총 생산량 비교 시나리오별 총 생산량을 막대 차트로 비교합니다
PPH 비교 시나리오별 시간당 1인 생산성을 막대 차트로 비교합니다
효율성 비교 (레이더) 로봇 가동률, 재봉기 가동률 등을 레이더 차트로 한눈에 비교합니다

9-3. 시나리오 관리

  • 각 시나리오 항목의 × 버튼으로 개별 삭제할 수 있습니다
  • [모두 삭제] 버튼으로 저장된 모든 시나리오를 한번에 삭제할 수 있습니다
  • 저장된 시나리오 목록에서 색상 표시로 각 시나리오를 구분할 수 있습니다
활용 예시: 로봇 속도를 75%, 85%, 100%로 각각 시뮬레이션한 뒤 시나리오를 저장하면, 속도 변화에 따른 생산성 차이를 한눈에 비교할 수 있습니다.

  • 파일명은 CoRobot_[모드]_Report_날짜_시분.pdf 형식으로 자동 생성됩니다
