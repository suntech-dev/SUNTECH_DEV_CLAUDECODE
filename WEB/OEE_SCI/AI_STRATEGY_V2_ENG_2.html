Current System Architecture (V1 Analysis)

Target Equipment

Real-time OEE data reception from computer pattern sewing machines at SCI Indonesia.

Real-time Communication

SSE-based 5-second polling. Equipment sends data via REST API to MySQL in real time.

OEE Calculation

Automatic calculation of Availability x Performance x Quality. Aggregated by time slot and shift with full history.

Alert System

3 event types: Andon, Downtime, Defective. Occurrence/completion handling with real-time dashboard display.

Export Report Feature

Clicking the Export Report button in the Signage header triggers a date selection modal. Choose from Today / Yesterday / Last 7 Days / Last 30 Days presets or a Custom Range, then download an HTML report in a new tab via proc/ai_report_export.php.

Note: AI reports use Rule-based analysis — no external LLM (e.g. Claude API). Insights are generated using statistical algorithms: moving average, Z-Score, and linear regression.

05

Detailed AI Feature Implementation

1

OEE Prediction (Predictive OEE)

Implemented

Applied to: AI Dashboard, OEE monitoring chart with forecast line

  • Analyzes hourly OEE patterns from the past 30 days
  • Linear Regression forecasts next 30 steps of OEE
  • Moving Average smooths short-term trends
  • Dashed forecast interval visualized on chart
Implementation: proc/ai_analysis.php — calls linearRegression($oee_series, 30)
2

Predictive Maintenance

Implemented

Applied to: AI Dashboard at-risk machine panel, Downtime monitoring with risk indicator column

  • Analyzes Downtime occurrence cycle patterns per machine
  • Compares elapsed time since last Downtime against average cycle
  • Calculates risk score (0~100) in real time

3

Anomaly Detection

Implemented

Applied to: AI Dashboard real-time alert panel, existing monitoring pages

  • Z-Score based: Instant detection when OEE, Defective rate, or Downtime frequency exceeds ±2σ
  • Pattern anomaly: Detects a drop of -15% or more in OEE compared to the previous hour
  • Cascading anomaly: Detects multiple machines failing simultaneously on the same line within 30 minutes
Implementation: proc/ai_analysis.php — Z-Score anomaly detection logic included
4

Quality Sentinel

Implemented

Applied to: AI Dashboard AI insights panel

  • Defective occurrence pattern analysis (time slot x machine x defective type)
  • Automated Pareto analysis: "80% of defectives originate from these 3 types"
  • Correlation analysis: predicts OEE impact when a specific Defective type increases
Implementation: proc/ai_analysis.php — quality analysis insights generation included
5

AI Report Export

Implemented

Applied to: ai_dashboard_2.php (Signage) header — Export Report button

  • Date selection modal → specify period → download HTML report in new tab
  • Automatically analyzes OEE, Downtime, and Defective data using Rule-based algorithms
  • Auto-generates an executive summary report (no Claude API — pure PHP statistical algorithms)
  • Includes OEE forecast, maintenance risk, anomaly detection, and AI insights sections
Implementation: proc/ai_report_engine.php (analysis engine) + proc/ai_report_export.php (HTML output)
6

Real-time Streaming AI Analysis

Implemented (Phase 2)

Applied to: AI Dashboard streaming AI analysis panel

  • SSE (Server-Sent Events) based real-time AI analysis stream
  • AI insights generated and updated in real time as data arrives
  • OEE change detection → immediate analysis result push
  • Uses shared stream library (stream_helper.lib.php)
Implementation: proc/ai_stream_analysis.php — SSE streaming AI analysis endpoint
7

Production Optimization Suggestions

Implemented (Phase 2)

Applied to: AI Dashboard optimization suggestions panel

  • Comprehensive analysis of production data per line and machine
  • Auto-generates specific optimization action items to improve OEE
  • Production schedule optimization suggestions based on Downtime patterns
  • Quality improvement recommendations based on Defective type analysis
Implementation: proc/ai_optimization.php — production optimization suggestions API
