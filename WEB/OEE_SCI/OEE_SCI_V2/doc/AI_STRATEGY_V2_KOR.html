01

현재 시스템 구조 (V1 분석)

대상 장비

인도네시아 SCI 컴퓨터 패턴재봉기의 실시간 OEE 수신

실시간 통신

Server-Sent Events (SSE) 기반 5초 폴링. 장비에서 REST API를 통해 MySQL로 실시간 데이터 수신.

OEE 계산

가용성 × 성능 × 품질 자동 계산. 시간대별/교대조별 집계 및 이력 보관.

알림 체계

Andon · Downtime · Defective 3종 이벤트. 발생/완료 처리 및 대시보드 실시간 표시.

항목 내용
기술 스택 PHP + MySQL + jQuery + Chart.js + SSE
UI 프레임워크 SAP Fiori Horizon Light
핵심 데이터 data_oee · data_oee_rows_hourly · Downtime · Defective · Andon
계층 구조 Factory → Line → Machine
현재 AI 기능 없음 (V1 Dashboard 이름만 "AI OEE")
02

AI 도입 핵심 방향

V1의 dashboard.php는 "AI OEE CS DASHBOARD"라는 이름을 갖고 있으나 실제 AI 기능이 전혀 없습니다. V2에서는 AI Dashboard를 별도 페이지로 신규 구현하고, 기존 페이지에도 AI 분석을 단계적으로 추가합니다.

예측 (Predictive)

과거 데이터 패턴으로 OEE, Downtime을 미래 예측. 사전 대응 가능.

Phase 1

감지 (Detection)

Z-Score 기반 실시간 이상 감지. 정상 범위 이탈 즉시 알림.

Phase 1

생성 (Generative)

AI 자동 보고서 생성. 요약 및 개선 권고 자동 작성.

04

AI 기능별 상세 설계

1

OEE 예측 (Predictive OEE)

Phase 1

적용 위치: AI Dashboard, OEE 모니터링 차트에 예측선 추가

  • 과거 30일 시간대별 OEE 패턴 학습
  • 요일 × 시간대 조합 매트릭스로 다음 4~8시간 OEE 예측
  • 차트에 점선 예측 구간(Confidence Interval) 시각화

활용 예시: "현재 오후 2시 OEE 72% → 다음 2시간 예상 68%로 하락 예측 (월요일 오후 패턴 기반)"

2

예측적 유지보수 (Predictive Maintenance)

Phase 1

적용 위치: AI Dashboard 위험 기계 패널, Downtime 모니터링에 위험 지표 열 추가

  • 머신별 Downtime 발생 주기 패턴 분석
  • 마지막 Downtime 이후 경과 시간 + 평균 주기 비교
  • 위험도 스코어 (0~100) 실시간 계산
스코어 등급 표시
80~100 위험 빨간색
50~79 주의 노란색
0~49 정상 초록색
3

이상 감지 (Anomaly Detection)

Phase 1

적용 위치: AI Dashboard 실시간 알림 패널, 기존 모니터링 페이지

  • Z-Score 기반: OEE · Defective율 · Downtime 빈도가 ±2σ 벗어날 시 즉시 감지
  • 패턴 이상: 이전 시간 대비 OEE -15% 이상 급락
  • 연쇄 이상: 동일 라인에서 30분 내 다수 머신 동시 문제 감지
4

AI 품질 감시 (Quality Sentinel)

Phase 1

적용 위치: Defective 모니터링에 AI 분석 탭 추가

  • Defective 발생 패턴 클러스터링 (시간대 × 머신 × Defective 유형)
  • 파레토 분석 자동화: "Defective의 80%가 이 3가지 유형에서 발생"
  • 상관관계 분석: 특정 Defective 유형 증가 → 이후 OEE 영향도 예측
5

AI 리포트 생성기

Phase 1

적용 위치: 보고서 페이지에 "AI 분석 보고서" 버튼 추가

  • 선택 기간의 OEE · Downtime · Defective 데이터를 AI로 자동 분석
  • 요약 보고서 자동 생성
