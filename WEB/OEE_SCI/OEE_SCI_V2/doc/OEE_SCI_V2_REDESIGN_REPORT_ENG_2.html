Background
Introduction & Motivation

OEE_SCI V1 provided stable real-time monitoring, but had no actual AI capabilities despite being named "AI OEE Dashboard."
V2 was designed to overcome the limitations of purely reactive monitoring.

⚠️ V1 Limitation — Reactive Only

Issues detectable only after they occur. Manual root-cause analysis for OEE drops. No proactive intervention capability.

🏭 Target Environment

SCI garment factory in Indonesia. Multiple computer pattern sewing machines. Existing SSE-based real-time data pipeline retained as-is.

🎯 V2 Core Objective

Deploy real AI capabilities — prediction, anomaly detection, and optimization — so problems are addressed before they occur. Computer Embroidery Machine expansion.

💡 Design Principles

Full backward compatibility with V1. Self-contained statistical engine (no external AI APIs). Optimized for 1920×1080 signage displays.

V1 Analysis
Existing System Architecture (V1)

The V1 system's components and technology stack, which served as the baseline for V2 design.

Target Equipment

Real-time OEE monitoring for computer pattern sewing machines at SCI Indonesia factory.

Real-time Communication

Server-Sent Events (SSE) with 5-second polling. Equipment → REST API → MySQL real-time data ingestion.

OEE Calculation

Automatic calculation of Availability × Performance × Quality. Hourly and shift-based aggregation with history retention.

Alert System

3 event types: Andon · Downtime · Defective. Event start/end processing with real-time dashboard display.

AI Intelligence
New AI Features — Detail

Real-time AI analysis powered by a self-built statistical engine (exponential smoothing, Z-Score, linear regression, Pearson correlation). No external AI APIs used.

NEW AI Intelligence Dashboard — Unified AI Hub (ai_dashboard_2.php)

A dedicated dashboard integrating 5 AI analysis sections in a single view. Auto-refreshes every 60 seconds with Factory→Line→Machine cascade filters and date range selection. Signage-optimized at 1920×1080.

  • OEE forecast summary card (next 4h prediction, trend direction badge ↑↓→)
  • At-risk machine count and active anomaly count in real time
  • Per-line AI Health Index (percentage bar + Normal/Caution/Danger grade)
  • Predictive maintenance recommendation list (sorted by risk score)
  • Real-time AI event streaming feed (slide-in animation)
  • Production optimization suggestion cards (P1/P2/P3 priority)
AI Intelligence Dashboard — Live Screen
📈

OEE Prediction (Predictive OEE)

Learns 30-day hourly OEE patterns to forecast OEE for the next 4–8 hours, with 90% confidence intervals (upper/lower bounds) and trend direction badges.

Exponential Smoothing + Day×Hour Seasonality
🚨

Anomaly Detection

Monitors per-machine OEE, Availability, and Quality metrics in real time. Instantly detects deviations beyond ±2σ mean or cascading anomalies within the same line.

Z-Score + Cascade Anomaly Detection
🔧

Predictive Maintenance Risk

Calculates a maintenance risk score (0–100) by weighting: runtime (40%) + downtime frequency increase rate (35%) + OEE instability (25%).

Weighted Scoring (3-Factor Composite)
📊

AI Quality Guardian

Auto-calculates defect type Pareto analysis (80% rule), 24-hour occurrence heatmap, per-machine risk ranking, and OEE Pearson correlation coefficient.

Pareto + Heatmap + Pearson Correlation
⚡

Real-time AI Streaming

SSE-based streaming of OEE anomalies, new downtime events, and maintenance risk alerts on 15-second cycles. Up to 15 slide-in card feed items displayed.

SSE + 5-min Session + Auto-reconnect
🎯

Production Optimization

Identifies bottleneck components (Availability/Performance/Quality) per line, estimates potential OEE improvement, and auto-generates 4 P1/P2/P3 improvement suggestions.

Bottleneck Analysis + Potential OEE Estimation

NEW 280CTP_IoT_INTEGRATED_V1_BLACK_CPU — PATTERN_MACHINE Dedicated (2026-03-17)

Separated from the V1 integrated firmware, this BLACK CPU board version is dedicated to PATTERN_MACHINE. SEWING_MACHINE (TrimPin ISR) and current sensor (ADC_SAR_Seq) are fully removed to simplify code and reduce Flash usage by ~5% compared to V1.

  • PATTERN_MACHINE fixed — machineType setting UI removed, CountFunc → PatternCountLoop fixed
  • SEWING_MACHINE fully removed — TrimPin ISR, SewingCountLoop(), makeAndonSewingCount2() removed
  • Current sensor fully removed — ADC_SAR_Seq TopDesign removed, currentSensor.c project source removed
  • UART JSON reception retained — {"cmd":"count","value":1,...} format → patternActualH/L accumulation
  • Flash layout compatible — COUNT / MACHINE_PARAMETER struct fields preserved (existing config data retained)

NEW 280CTP_IoT_INTEGRATED_V1_EMBROIDERY_S — Computer Embroidery Machine Dedicated (2026-03-26)

Based on BLACK_CPU V1, this version is dedicated to Computer Embroidery Machine. The UART protocol is completely replaced from JSON to semicolon-delimited text format, with thread breakage tracking and embroidery-specific server API added.

  • Semicolon parser newly implemented — actual_qty;cycle_time_ms;thread_breakage_qty;motor_runtime_ms; format, parsing starts on 4th ';' detection
  • Thread breakage tracking — embThreadBreakageQty/SumH/SumL fields added, tb item displayed on LCD Page 1
  • Embroidery-specific WebAPI — send_eCount.php → data_embroidery table, ms→seconds (1 decimal) conversion
  • UART TEST viewer — last 5 received packets mirrored on LCD (uartTestMenu.c newly created)
  • Echo loopback bug fix — non-numeric bytes ignored at index=0, DBG printf removed (2026-03-27)
