Latar Belakang
Pendahuluan & Motivasi

OEE_SCI V1 menyediakan pemantauan real-time yang stabil, namun tidak memiliki kemampuan AI nyata meskipun dinamai "AI OEE Dashboard."
V2 dirancang untuk mengatasi keterbatasan pemantauan yang sepenuhnya reaktif.

⚠️ Keterbatasan V1 — Hanya Reaktif

Masalah hanya dapat terdeteksi setelah terjadi. Analisis akar penyebab penurunan OEE dilakukan secara manual. Tidak ada kemampuan intervensi proaktif.

🏭 Lingkungan Target

Pabrik garmen SCI di Indonesia. Beberapa mesin jahit pola komputer. Pipeline data real-time berbasis SSE yang ada dipertahankan sepenuhnya.

🎯 Tujuan Inti V2

Menerapkan kemampuan AI nyata — prediksi, deteksi anomali, dan optimasi — sehingga masalah dapat ditangani sebelum terjadi. Perluasan Mesin Bordir Komputer.

💡 Prinsip Desain

Kompatibilitas penuh ke belakang dengan V1. Mesin statistik mandiri (tanpa API AI eksternal). Dioptimalkan untuk tampilan signage 1920×1080.

Peralatan Target

Pemantauan OEE real-time untuk mesin jahit pola komputer di pabrik SCI Indonesia.

Komunikasi Real-time

Server-Sent Events (SSE) dengan polling 5 detik. Ingesti data real-time: Peralatan → REST API → MySQL.

Kalkulasi otomatis Availability × Performance × Quality. Agregasi per jam dan per shift dengan retensi riwayat.

Sistem Alert

3 jenis event: Andon · Downtime · Cacat. Pemrosesan mulai/akhir event dengan tampilan dasbor real-time.

BARU 280CTP_IoT_INTEGRATED_V1_BLACK_CPU — Khusus PATTERN_MACHINE (2026-03-17)

Dipisahkan dari firmware terpadu V1, versi board BLACK CPU ini didedikasikan untuk PATTERN_MACHINE. SEWING_MACHINE (TrimPin ISR) dan sensor arus (ADC_SAR_Seq) dihapus sepenuhnya untuk menyederhanakan kode dan mengurangi penggunaan Flash ~5% dibandingkan V1.

  • PATTERN_MACHINE ditetapkan — UI pengaturan machineType dihapus, CountFunc → PatternCountLoop diperbaiki
  • SEWING_MACHINE dihapus sepenuhnya — TrimPin ISR, SewingCountLoop(), makeAndonSewingCount2() dihapus
  • Sensor arus dihapus sepenuhnya — ADC_SAR_Seq TopDesign dihapus, sumber proyek currentSensor.c dihapus
  • Penerimaan JSON UART dipertahankan — format {"cmd":"count","value":1,...} → akumulasi patternActualH/L
  • Tata letak Flash kompatibel — field struct COUNT / MACHINE_PARAMETER dipertahankan (data konfigurasi yang ada tetap)

BARU 280CTP_IoT_INTEGRATED_V1_EMBROIDERY_S — Khusus Mesin Bordir Komputer (2026-03-26)
