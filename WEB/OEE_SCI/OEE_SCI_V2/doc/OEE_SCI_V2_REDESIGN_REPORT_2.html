Background
AI 도입 배경

OEE SCI V1은 안정적인 실시간 모니터링을 제공했지만, 이름만 "AI OEE Dashboard"였을 뿐 실질적인 AI 기능이 전혀 없었습니다.
사전 대응이 불가능한 수동 분석 구조의 한계를 극복하기 위해 V2를 기획하였습니다.

⚠️ V1의 한계 — 반응적 모니터링

문제가 발생한 후에야 확인 가능. OEE 저하 원인 분석이 수동. 이상 발생 시 즉각 대응 불가.

🏭 대상 환경

인도네시아 SCI. 컴퓨터 패턴재봉기 대상. SSE 기반 실시간 데이터 수신 구조 기존 유지.

🎯 V2 핵심 목표

실제 동작하는 AI 기능을 탑재하여, 문제 발생 전 예측·감지·최적화가 가능한 시스템으로 진화. 컴퓨터 자수기 확장.

💡 설계 원칙

V1 기능 완전 유지. 외부 AI API 없이 자체 통계 엔진으로 구현. 1920×1080 사이니지 최적화.

과거 30일 시간대별 OEE 패턴을 학습하여 다음 4~8시간 OEE를 예측. 사전 대응 가능.

✓ 구현

📝 생성 (Generative)

규칙 기반 AI 자동 보고서 생성. 경영진용 OEE 요약 · 이상감지 · 정비위험 · 최적화 내용을 자연어로 자동 작성.

✓ 구현

⚡ 스트리밍 AI (Streaming)

SSE 기반 실시간 AI 분석 스트림. 데이터 입수와 동시에 AI 인사이트를 슬라이드인 카드 피드로 표시.

✓ 구현

🎯 최적화 (Optimization)

라인별 Availability · Performance · Quality 병목을 식별하고 P1/P2/P3 개선 제안 자동 산출.

✓ 구현

📈

OEE 예측 (Predictive OEE)

과거 30일 시간대별 OEE 패턴을 학습하여 다음 4~8시간 OEE를 예측합니다. 90% 신뢰구간(상한/하한)과 트렌드 방향 뱃지를 함께 표시합니다.

지수평활법 + 요일×시간 계절성
🚨

이상 감지 (Anomaly Detection)

머신별 OEE·가용성·품질 지표를 실시간 모니터링합니다. 평균 ±2σ를 벗어나거나 동일 라인에서 연쇄 이상이 발생하면 즉시 감지합니다.

Z-Score + 연쇄 이상 감지
🔧

예방정비 위험도

런타임(40%) + 다운타임 빈도 증가율(35%) + OEE 불안정도(25%)를 가중 합산해 정비 위험 스코어(0~100)를 계산합니다.

가중 스코어링 (3요소 합산)
📊

AI 품질 파수꾼

불량 유형 파레토 분석(80% 룰), 24시간 발생 히트맵, 머신별 위험 랭킹, OEE 피어슨 상관계수를 자동 산출합니다.

파레토 + 히트맵 + Pearson
⚡

실시간 AI 스트리밍

SSE 기반으로 OEE 이상·신규 다운타임·정비 위험 이벤트를 15초 주기로 스트리밍합니다. 최대 15건 슬라이드인 카드 피드로 표시됩니다.

SSE + 5분 세션 + 자동 재연결
🎯

생산 최적화 제안

라인별 Availability·Performance·Quality 병목 컴포넌트를 식별하고, 잠재 OEE 향상 추정치와 P1/P2/P3 개선 제안 4건을 자동 생성합니다.
