6 Core AI Features Guide

A signage-style dashboard that uses AI to analyze, predict, and optimize OEE (Overall Equipment Effectiveness) on the manufacturing floor. Displays 6 AI features simultaneously on a 1920×1080 screen in real-time, with drill-down filters at factory, line, and machine level.

Feature 01

OEE Trend & AI Forecast

Actual OEE trend + AI-based 4-hour forecast chart

OEE (Overall Equipment Effectiveness) is the core manufacturing KPI calculated as Availability × Performance × Quality. This chart displays today's actual OEE history (solid line) and AI-computed 4-hour forecast (dashed line) on a single screen.

The semi-transparent band around the forecast line is the 90% Confidence Interval (CI), meaning there is a 90% probability that the actual OEE will fall within this range. A wider CI indicates higher uncertainty; a narrower CI indicates higher forecast reliability.

Actual OEE (Solid Line)

Actual OEE values by hour for today. Always shown based on the current date.

AI Forecast (Dashed Line)

Predicted OEE for up to 4 hours from now. Uses Exponential Smoothing + seasonal patterns.

Confidence Interval (90% CI)

Shows uncertainty in the forecast range. A narrower interval means higher prediction accuracy.

⚙ AI Forecast Algorithm — Exponential Smoothing + Seasonal Patterns
  • STEP 1 — Seasonal Learning: Learns OEE statistics by hour and day-of-week for the past 7 or 30 days, building a seasonality matrix (day-of-week × hour).
  • STEP 2 — Today's Actuals Collection: Retrieves actual OEE by hour for today (always based on today's date regardless of the filter selected).
  • STEP 3 — Forecast Calculation: Combines Exponential Smoothing with seasonality-weighted averages to predict OEE by hour for the next 4 hours.
  • STEP 4 — Confidence Interval Computation: Calculates 90% CI based on standard deviation (seasonal_std capped at 15% to prevent extreme intervals).
  • STEP 5 — Trend Assessment: Analyzes the slope of forecast values using linear regression and displays Trending Up / Down / Stable badges.

Top Summary Cards

🔴 Real-time OEE (LIVE)

The actual OEE value for the most recent hour as of now.
Even when the date filter is set to 7 or 30 days, this value is always based on today.

Status badges: GOOD  WARNING  CRITICAL

🔮 Next 4H AI Forecast

Displays the average predicted OEE for the next 4 hours and the CI range.
Example: "78.3% · CI: 72.0% ~ 84.6%"

Trend badges: Trending Up ↑  Trending Down ↓  Stable →

Does changing the date range affect the forecast values?
Yes, it does. The date range filter affects the period used for pattern learning. For example, selecting '7 Days' forecasts based on OEE patterns from the past 7 days by hour, while '30 Days' uses a longer 30-day dataset for learning. However, the current OEE (Real-time) and today's actual OEE line are always fixed to today's date.
What does it mean when the confidence interval (CI) is very wide?
A wide CI means that the OEE variability during that time period is high. The forecast is for reference only, and the actual value is likely somewhere within that range. The system limits the standard deviation cap to 15% to prevent the CI from becoming abnormally wide.

Feature 02

Anomaly Detection

Z-Score based real-time anomaly detection

Anomaly Detection statistically measures how far the current OEE deviates from historical patterns and automatically identifies abnormal equipment conditions.

It uses the statistical method known as the Z-Score (standard score), which represents how many standard deviations the current value is from the mean. A higher Z-Score means the current value has deviated significantly from the normal range, indicating an anomalous state.

⚙ Z-Score Anomaly Detection Principle
  • Z = (Current Value − Mean) / Standard Deviation
  • Compares the current hour's OEE to the historical statistics for the same time slot (day-of-week and hour).
  • If the Z-Score exceeds a threshold, the event is flagged as an anomaly and added to the list.
  • Severity (Critical / Warning) is differentiated by color and badge.

Anomaly Severity Criteria

Severity Condition Meaning Display Color
Critical Very high Z-Score (severe deviation) Requires immediate inspection and action Red
Warning High Z-Score (caution-level deviation) Trend monitoring and root cause analysis recommended Yellow
Normal Low Z-Score (within normal range) Normal operating state Green
🔴 Top Card — Anomaly Detection

Displays the total number of anomalies currently detected as a number.
Shows "0 anomalies" when no anomalies are present.
A blinking red badge appears when Critical anomalies are detected.

📋 Anomaly Detection List Card

Lists the machine name, line name, anomaly timestamp, and Z-Score for each detected anomaly.
Displays "No anomalies detected" when none are found.

What data is used as the baseline for anomaly detection?
The system calculates the mean and standard deviation of OEE by hour for the same day-of-week and time slot, based on the selected date range (Today / 7 Days / 30 Days). The Z-Score is then computed by comparing the current OEE to these baseline values.
Does 0 anomalies mean everything is fully normal?
It means no data has exceeded the Z-Score anomaly threshold — statistically within the normal range. However, even if the absolute OEE value is low, it may not be flagged as an anomaly if the variation pattern is consistent. Always check the OEE Trend chart alongside this indicator.

Feature 03

Line Health Index

Equipment health score per line based on average OEE

The Line Health Index quantifies the overall equipment condition of each production line on a 0–100% scale. It is derived from the average OEE, where higher values indicate healthier line operation.

The analysis period changes based on the selected date range. For example, selecting "Last 7 Days" calculates the health score using the average OEE by line over the past 7 days. The card subtitle updates automatically to reflect the chosen period.

📊 Top Card — Line Health Index (Avg)

Displays the average health score across all lines as a large number.
Provides an at-a-glance view of overall factory health status.

💡

Note: The Line Health Index uses OEE values directly as the health score. It visually shows how closely each line is achieving the OEE target (85%). Use it together with the Production Optimization feature to prioritize improvement efforts.

If the date filter is set to 'Today', is only today's data used?
When 'Today' is selected, the system internally uses the past 7 days for statistical stability. A single day's data is insufficient to compute a reliable average. When '7 Days' or '30 Days' is selected, the actual line-level OEE average for that period is used.
What should I do when a line's health score is low?
Lines with a low health score are also targeted as improvement opportunities in the Production Optimization feature. To identify which component (Availability / Performance / Quality) is causing the low score, use the bottleneck analysis in Production Optimization, and check the high-risk machines on that line using Predictive Maintenance.

Feature 04

Predictive Maintenance

Machine-level failure prediction and maintenance priority via risk score

Predictive Maintenance analyzes historical downtime records and OEE stability data to calculate a failure risk score (0–100) for each machine.

By identifying high-risk machines in advance and scheduling planned maintenance, production losses from unexpected failures can be proactively prevented.

Risk Score Calculation (Weighted Sum of 3 Indicators)

A
Runtime Risk — Weight 40%

Compares the elapsed time since the last downtime to the historical average failure interval. The closer the elapsed time is to the average interval, the higher the risk score.
Formula: Elapsed Time (h) ÷ Average Failure Interval (h)

B
Downtime Frequency Increase Rate — Weight 35%

Compares downtime occurrences in the most recent 30 days to the prior 30 days (31–60 days ago). Machines with a sharp increase in recent downtime frequency are flagged as failure indicators.
Formula: (Recent 30-day count − Prior 30-day count) ÷ Prior 30-day count

C
OEE Instability — Weight 25%

Analyzes the average and variability (standard deviation) of OEE per machine over the analysis period. Low OEE with high variability is considered an unstable equipment condition.
Note: OEE values exceeding 100% are clamped to 0–100% to prevent false positives.

Risk Grade Criteria

Risk Grade Score Range Meaning Recommended Action
Danger 🔴 80 or above Very high probability of failure Immediate inspection and maintenance
Warning 🟡 50–79 Failure symptoms detected, caution needed Plan early inspection
Normal 🟢 0–49 Normal operating state Maintain routine monitoring
🔴 Top Card — High-Risk Machines

Displays the count of machines in Danger grade (score ≥ 80).
The number of Warning-grade machines is also shown with a badge.

📋 Predictive Maintenance List

Machines are sorted by risk score in descending order.
Machine name, line name, risk score, and risk grade are all displayed.

How often is the risk score updated?
The risk score is recalculated every 60 seconds, in sync with the dashboard's auto-refresh cycle. To update immediately, click the 'Refresh' button at the top.
How are new machines with no downtime history handled?
Machines with no downtime history receive a score of 0 for Runtime Risk (A) and Frequency Increase Rate (B). Only the OEE Instability (C) component is calculated. After 90 days of data accumulation, more accurate risk predictions become possible.
Can actual maintenance schedules be integrated with this system?
The current version focuses on risk score delivery and alert functionality. Integration with maintenance scheduling systems (ERP / CMMS) is available as a custom development option.

Feature 05

Real-time AI Streaming

Live streaming feed for anomaly, downtime, and maintenance risk events

Real-time AI Streaming displays all AI analysis results from the dashboard as an event feed in real time. Anomaly detection, downtime occurrences, and maintenance risk events appear in the bottom feed as soon as they happen.

It uses SSE (Server-Sent Events) technology to maintain a persistent connection with the server and push new events to the feed in real time. Connection status can be monitored via the blinking dot at the top.

Streaming Event Types

Event Type Color Description Example
Anomaly Red Z-Score based OEE anomaly detection event Line A-1 / Machine M-03 anomaly detected (Z-Score: 3.2)
Downtime Yellow Equipment downtime occurrence event Line B-2 / Machine M-07 downtime (12 min)
Maintenance Orange Predictive maintenance risk escalation event Machine M-11 risk score 85 — immediate inspection advised
Connection Status

Green blinking = Connected
Orange blinking = Connecting
Red blinking = Disconnected

Event Counter

The total number of events received in the current session is displayed in real time.
Example: "47 events"

Feed Display Mode

The latest events are prepended to the top, and older events are automatically removed once the count exceeds a limit.

💡

Usage Tip: The Real-time AI Streaming feed acts as the source event stream for the Anomaly Detection and Predictive Maintenance cards. When the numbers on the top cards increase, you can immediately identify which machine or line triggered the event in this feed.

Are events lost when the connection drops?
If the SSE connection drops, the system automatically attempts to reconnect. Some events may be missed during reconnection, but the main dashboard data (anomaly list, maintenance scores, etc.) is refreshed via a separate 60-second polling cycle, so overall status visibility is not affected.
How long is streaming data retained?
The Real-time AI Streaming feed is only maintained for the current screen session (browser connection duration). Refreshing the page resets the feed. For historical event lookup, please use the separate data query page.

Feature 06

Production Optimization

OEE bottleneck analysis and AI-driven production improvement suggestions

Production Optimization automatically identifies lines falling below the OEE target (85%), then uses AI to analyze which of the three components — Availability, Performance, or Quality — is the biggest bottleneck (loss factor), and provides specific improvement suggestions.

Rather than simply reporting that OEE is low, it answers the question: "Where should we improve first for the greatest effect?" — backed by data.

⚙ Production Optimization Analysis Process
  • STEP 1: Compute average OEE, Availability, Performance, and Quality by line for the selected period (7 / 14 / 30 days)
  • STEP 2: Filter only lines falling below the OEE target (85%) for analysis
  • STEP 3: Calculate gaps — Availability gap (target 90%), Performance gap (target 90%), Quality gap (target 99%)
  • STEP 4: The largest gap among the three becomes the bottleneck factor
  • STEP 5: Automatically generate improvement suggestions (Suggestions) based on the bottleneck factor
  • STEP 6: Compare OEE between the first half and second half of the period to determine trend (Improving / Declining / Stable)

OEE Targets and Gap Analysis Criteria

📖

Source of the 85% Benchmark — TPM World-Class OEE (Seiichi Nakajima, 1988)
The 85% OEE target is based on the World-Class OEE standard established by Seiichi Nakajima, the creator of TPM (Total Productive Maintenance). It is derived from: Availability 90% × Performance 95% × Quality 99.9% ≈ 85.4%. This benchmark is also referenced by international standards including ISO 22400 and SMRP (Society for Maintenance & Reliability Professionals). This system applies these criteria while relaxing the Performance target to 90% to better reflect realistic conditions on the shop floor.

Metric Target Meaning of Large Gap Key Improvement Direction
OEE 85% Overall equipment efficiency decline Focus on bottleneck among Availability, Performance, and Quality
Availability 90% Excessive downtime or setup time Strengthen preventive maintenance, reduce setup time, apply SMED
Performance 90% Equipment speed loss or frequent minor stops Analyze minor stoppage causes, optimize equipment speed
Quality 99% Defective products or rework occurring Review process parameters, check raw material quality

Improvement Priority Classification

P1 — Urgent

OEE gap of 20%p or more
Immediate corrective action required

P2 — Important

OEE gap of 10–20%p
Short-term improvement plan recommended

P3 — Monitor

OEE gap of less than 10%p
Ongoing monitoring and incremental improvement

📊 Optimization Summary Bar

Displays key summary metrics in the card header: total lines, lines below target, overall average OEE, and the top-performing line.

📋 Optimization Opportunity List

Under-performing lines are listed in ascending OEE order, with the bottleneck factor, gap, and improvement suggestions shown per line.

What is displayed when all lines meet the 85% target?
When all lines are meeting the OEE target (85%), the message "All lines are meeting the target OEE" is shown along with the current average OEE and the highest OEE value. This indicates a target-exceeded state.
How is the trend (Improving / Declining / Stable) calculated?
The analysis period is split into two halves, and the OEE average of each half is compared. For example, with a 14-day analysis, the average OEE of the most recent 7 days is compared to the prior 7 days. If the recent half is higher: Improving, if lower: Declining, and if the difference is minimal: Stable.
What is the basis for the improvement suggestions?
Based on the bottleneck factor (Availability / Performance / Quality), predefined improvement suggestion texts derived from manufacturing best practices are automatically selected. For example, if Availability is the bottleneck, suggestions such as "Optimize preventive maintenance intervals" or "Review setup time reduction" are recommended. The current version uses rule-based suggestions; future integration with generative AI will enable more detailed and tailored recommendations.

FAQ

Frequently Asked Questions

Key Q&A covering the dashboard overall
Does the AI work with limited data?
It works, but accuracy may be lower. OEE forecasting requires at least a few days of hourly data, and predictive maintenance risk becomes more accurate as 90 days of downtime history accumulates. During initial deployment, treat outputs as reference indicators; reliability improves after 1–3 months of operation.
Does the factory / line / machine filter apply to all 6 AI features?
Yes, changing the factory / line / machine filter at the top applies the filter across all 6 AI features. However, the Real-time OEE card and today's OEE Trend line are always fixed to the current date.
What does the Export button do?
It exports the current dashboard's key metrics and analysis results as a PDF or image file. Clicking the Export button opens an export modal where you can select the language (Korean / English) before generating the file.
How often does the dashboard auto-refresh?
All AI analysis data refreshes automatically every 60 seconds (1 minute). To refresh immediately, click the 'Refresh' button at the top. Real-time AI Streaming receives events separately via SSE independently of this cycle.
Is it suitable for signage displays (TV monitors)?
This dashboard is designed specifically for 1920×1080 signage. It is recommended to display it on a 4K / FHD TV using full-screen mode (F11). The dark theme ensures high readability on large monitors in dimly lit factory environments.
