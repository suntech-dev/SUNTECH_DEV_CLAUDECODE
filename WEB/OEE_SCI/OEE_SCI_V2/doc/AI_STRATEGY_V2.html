01

현재 시스템 구조 (V1 분석)

🏭

대상 장비

인도네시아 봉제 공장의 패턴재봉기(IoT). MAC 주소 기반 자동 등록 및 실시간 생산량 수신.

⚡

실시간 통신

Server-Sent Events (SSE) 기반 5초 폴링. 장비 → REST API → MySQL → 브라우저.

📊

OEE 계산

가용성 × 성능 × 품질 자동 계산. 시간대별/교대조별 집계 및 이력 보관.

🔔

알림 체계

안돈 · 비가동 · 불량 3종 이벤트. 발생/완료 처리 및 대시보드 실시간 표시.

항목 내용
기술 스택 PHP + MySQL + jQuery + Chart.js + SSE
UI 프레임워크 SAP Fiori Horizon Light
핵심 데이터 data_oee · data_oee_rows_hourly · Downtime · Defective · Andon
계층 구조 Factory → Line → Machine
AI 현황 없음 (V1 Dashboard 이름만 "AI OEE")
02

AI 도입 핵심 방향

V1의 dashboard.php는 "AI OEE CS DASHBOARD"라는 이름을 갖고 있으나 실제 AI 기능이 전혀 없습니다. V2에서는 진짜 AI Dashboard를 별도 페이지로 신규 구현하고, 기존 페이지에도 AI 분석을 단계적으로 추가합니다.

🔮

예측 (Predictive)

과거 데이터 패턴으로 OEE, 다운타임을 미래 예측. 사전 대응 가능.

Phase 1
🔍

감지 (Detection)

Z-Score 기반 실시간 이상 감지. 정상 범위 이탈 즉시 알림.

Phase 1
🤖

대화 (Conversational)

Claude API 연동 자연어 어시스턴트. 공장 데이터를 한국어로 질문/답변.

Phase 2
📝

생성 (Generative)

AI 자동 보고서 생성. 경영진용 요약 및 개선 권고 자동 작성.

Phase 2

04

AI 기능별 상세 설계

1

OEE 예측 (Predictive OEE)

Phase 1

적용 위치: AI Dashboard, data_oee.php 차트에 예측선 추가

  • data_oee_rows_hourly 테이블에서 과거 30일 시간대별 OEE 패턴 학습
  • 요일 × 시간대 조합 매트릭스로 다음 4~8시간 OEE 예측
  • Chart.js에 점선 예측 구간(Confidence Interval) 시각화

알고리즘: 지수평활법(Exponential Smoothing) + 요일 계절성 보정

활용 예시: "현재 오후 2시 OEE 72% → 다음 2시간 예상 68%로 하락 예측 (월요일 오후 패턴 기반)"

ai_oee_prediction.php statistics.lib.php
2

예측적 유지보수 (Predictive Maintenance)

Phase 1

적용 위치: AI Dashboard 위험 기계 패널, data_downtime.php에 위험 지표 열 추가

  • 머신별 다운타임 발생 주기 패턴 분석
  • 마지막 다운타임 이후 경과 시간 + 평균 주기 비교
  • 위험도 스코어 (0~100) 실시간 계산
스코어 등급 표시
80~100 위험 빨간색
50~79 주의 노란색
0~49 정상 초록색
ai_maintenance.php statistics.lib.php
3

이상 감지 (Anomaly Detection)

Phase 1

적용 위치: AI Dashboard 실시간 알림 패널, 기존 모니터링 페이지

  • Z-Score 기반: OEE·불량률·다운타임 빈도가 ±2σ 벗어날 시 즉시 감지
  • 패턴 이상: 이전 시간 대비 OEE -15% 이상 급락
  • 연쇄 이상: 동일 라인에서 30분 내 다수 머신 동시 문제
ai_anomaly.php statistics.lib.php
4

AI 어시스턴트 (Claude API 연동)

Phase 2 · 핵심

적용 위치: AI Dashboard 하단 Q&A 패널

  • 운영자가 자연어로 질문 입력
  • PHP 백엔드가 관련 DB 데이터 자동 수집
  • Claude API에 데이터 컨텍스트 + 질문 전송
  • 한국어 자연어 답변 반환 및 화면 표시
질문 예시
"오늘 어느 라인이 가장 문제가 많나요?"
"이번 주 불량 원인을 분석해줘"
"다음 교대 OEE를 예측해줘"
"예방정비가 필요한 기계 순위를 알려줘"
api/ai/chat.php lib/ai_helper.lib.php claude-sonnet-4-6
5

AI 품질 파수꾼 (Quality Sentinel)

Phase 2

적용 위치: data_defective.php에 AI 분석 탭 추가

  • 불량 발생 패턴 클러스터링 (시간대 × 머신 × 불량유형)
  • 파레토 분석 자동화: "불량의 80%가 이 3가지 유형에서 발생"
  • 상관관계 분석: 특정 불량 유형 증가 → 이후 OEE 영향도 예측
api/ai/quality_analysis.php
6

AI 리포트 생성기

Phase 2

적용 위치: report.php에 "AI 분석 보고서" 버튼 추가

  • 선택 기간 데이터를 Claude API로 전송
  • 경영자용 요약 보고서 자동 생성 (한국어)
  • 엑셀 내보내기 시 AI 코멘트 시트 자동 포함
lib/ai_helper.lib.php claude-sonnet-4-6
