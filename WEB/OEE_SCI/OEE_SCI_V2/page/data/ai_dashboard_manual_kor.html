개요

AI 인텔리전스 대시보드는 AI/ML 알고리즘을 활용한 실시간 공장 모니터링 도구입니다. 생산 관리자가 문제를 조기에 발견하고, 기계 고장을 예방하며, OEE(종합 설비 효율)를 최적화할 수 있도록 지원합니다.

OEE란 무엇인가요?
OEE는 생산 라인이 얼마나 효율적으로 운영되는지를 측정하는 지표입니다.
OEE = 가용률(Availability) × 성능률(Performance) × 품질률(Quality)
85% 이상이면 세계 수준(World-class)으로 평가됩니다.

2

필터 사용법

대시보드 상단의 필터를 사용하면 특정 범위의 데이터만 조회할 수 있습니다. 필터를 변경하면 모든 섹션이 동시에 갱신됩니다.

  1. 전체 공장 (All Factory) — 전체 공장 데이터를 표시합니다. 특정 공장을 선택하면 해당 공장 데이터만 조회됩니다.
  2. 전체 라인 (All Line) — 공장 선택 후 활성화됩니다. 특정 생산 라인을 선택할 수 있습니다.
  3. 전체 설비 (All Machine) — 라인 선택 후 활성화됩니다. 특정 설비를 선택할 수 있습니다.
  4. 날짜 범위 (Today / Yesterday / Last 7 Days / Last 30 Days) — 분석 기간을 선택합니다. 선택 즉시 모든 AI 섹션이 해당 기간 기준으로 갱신됩니다.
날짜 범위 옵션 기준 권장 용도
Today 오늘 하루 당일 실시간 모니터링 (기본값)
Yesterday 전일 전날 마감 후 이상 여부 확인
Last 7 Days 최근 7일 주간 트렌드 분석
Last 30 Days 최근 30일 월간 성과 및 장기 패턴 분석
팁: Refresh 버튼을 클릭하면 필터 변경 없이 모든 섹션의 최신 데이터를 즉시 불러옵니다.
참고: 필터를 변경하면 실시간 스트리밍 피드(섹션 8)가 자동으로 재연결됩니다. 잠시 "연결 중..." 상태가 표시될 수 있으며, 이는 정상적인 동작입니다.

내비게이션 메뉴 (햄버거 버튼)

헤더 좌측 상단의 ☰ (햄버거) 버튼을 클릭하면 좌측 드로어 메뉴가 열립니다. Setting·Monitoring·Report 등 모든 관리 페이지로 이동할 수 있습니다.

Export (리포트 내보내기)

헤더의 Export 버튼을 클릭하면 분석 리포트를 내보낼 수 있습니다.

  1. Export 버튼 클릭 → 기간 선택 모달 표시
  2. 프리셋 버튼(Today / Yesterday / Last 7 Days / Last 30 Days) 또는 직접 날짜 범위 입력
  3. Export 클릭 → 새 탭에서 PDF 리포트 생성

요약 카드

상단의 4개 카드는 AI가 모니터링하는 핵심 지표를 한눈에 보여줍니다.

향후 4시간 OEE 예측

현재 OEE 값과 AI가 예측한 향후 4시간의 OEE를 표시합니다. 트렌드 배지(상승/하락/유지)도 함께 표시됩니다.

이상 감지

오늘 OEE가 통계적으로 비정상적인 생산 라인의 수입니다. 고위험 이상이 발생하면 빨간색 "CRITICAL" 배지가 표시됩니다.

고위험 설비 (High-Risk Machines)

DANGER(위험, 점수 70+) + CAUTION(주의, 점수 40~69) 설비의 합계입니다. 서브 텍스트에 각 수준별 개수가 표시되며, 배지는 DANGER가 있으면 빨간색, CAUTION만 있으면 노란색으로 구분됩니다.

라인 건강지수 (평균)

전체 라인의 평균 건강 점수 (0~100점). 7일 OEE 평균 기반. 70 미만이면 점검이 필요합니다.

트렌드 배지 의미
Improving (상승) OEE가 상승 추세 — 예측값이 현재값보다 높음
Stable (유지) OEE가 안정적 — 큰 변화 없이 유지될 것으로 예상
Declining (하락) OEE가 하락 추세 — 조치가 필요할 수 있음
4

OEE 트렌드 & AI 예측

과거 OEE 데이터와 AI가 생성한 향후 예측값을 함께 보여주는 차트입니다.

차트 요소 설명
실선 (파란색) 실제 측정된 OEE 수치 (과거 데이터)
점선 (주황색) AI 예측값 — 향후 OEE 예상치
음영 영역 90% 신뢰구간 — AI가 90% 확률로 실제값이 이 범위 안에 들어올 것으로 예측
읽는 방법: 점선 예측값이 85% 목표선 아래로 향하고 있다면, 실제 OEE가 떨어지기 전에 원인을 조사하는 것이 좋습니다.
예측 정확도 안내: AI 예측은 과거 데이터의 통계적 패턴을 기반으로 합니다. 갑작스러운 기계 고장, 자재 부족 등 예기치 못한 사건은 예측에 반영되지 않습니다.
5

이상 감지

오늘 OEE가 해당 라인의 과거 데이터(최근 30일)와 비교했을 때 통계적으로 비정상적인 생산 라인을 표시합니다.

이상 감지 방법: AI는 매 시간 각 라인의 OEE에 대한 Z-Score를 계산합니다. Z-Score가 2.0 이상이면 현재 OEE가 30일 평균에서 2표준편차 이상 낮다는 의미로, 통계적으로 유의미한 하락입니다.
심각도 Z-Score 범위 의미 권장 조치
DANGER (위험) Z ≥ 3.0 극단적인 OEE 하락 — 해당 라인에서 매우 드문 상황 즉시 조사 필요
WARNING (경고) 2.0 ≤ Z < 3.0 유의미한 OEE 하락 — 정상 범위 이탈 30분 이내 확인
이상 항목이 없다면? 좋은 신호입니다. 오늘 모든 라인이 정상 범위 내에서 운영되고 있습니다.
6

라인 건강지수

건강지수는 최근 7일간의 OEE 데이터를 기반으로 각 생산 라인의 전체 상태를 0~100점으로 요약합니다.

점수 상태 해석
80 – 100 양호 (Healthy) 라인이 잘 운영되고 있음 — 별도 조치 불필요
60 – 79 보통 (Fair) 성능 저하 징후 — 지속 모니터링 필요
0 – 59 불량 (Poor) 심각한 성능 문제 — 우선적으로 조사 필요
계산 방법: 건강지수 = 해당 라인의 7일 평균 OEE를 0~100 지수로 환산. OEE가 꾸준히 높은 라인은 높은 점수를, OEE 하락이나 다운타임이 잦은 라인은 낮은 점수를 받습니다.
7

예방정비 권고

고장 발생 전에 예방정비를 실시할 수 있도록, AI가 설비별 고장 위험 점수를 계산하고 높은 순서로 표시합니다.

위험 점수 위험 수준 권장 조치
70 – 100 고위험 (High Risk) 24시간 이내 정비 일정 수립
40 – 69 중위험 (Medium Risk) 이번 주 내 정비 계획 수립
0 – 39 저위험 (Low Risk) 정기 정비 일정 유지
위험 점수 계산 방식: AI가 3가지 요소를 종합합니다.
  • 다운타임 빈도 (40%): 최근 7일간 발생한 다운타임 이벤트 수
  • 다운타임 비율 (35%): 가동 가능 시간 대비 평균 다운타임 시간
  • 설비 노후도 (25%): 마지막 정비 기록 이후 경과 시간
중요: 위험 점수는 과거 패턴 기반의 예측 추정치입니다. 현장 점검 및 운전원의 경험과 함께 참고 자료로 활용하세요.

실시간 AI 스트리밍 분석

AI가 감지한 이벤트를 실시간으로 표시하는 라이브 피드입니다. 페이지를 새로 고침하지 않아도 자동으로 이벤트가 표시됩니다. 이벤트는 최신 순서(위에서 아래)로 표시됩니다.

이벤트 유형 아이콘 의미
OEE Anomaly Detected ⚠️ 오늘 생산 라인의 OEE가 과거 평균보다 유의미하게 하락함
Active Downtime 🔴 설비가 현재 비계획 다운타임 상태 (미해결)
Maintenance Risk Alert 🔧 설비 위험 점수가 고위험 임계값 초과
Quality Alert 🔬 라인의 품질률이 허용 기준 이하로 하락
심각도 배지 색상 의미
DANGER 빨간색 즉시 조치 필요
WARNING 주황색 빠른 시일 내 검토 필요
INFO 파란색 정보성 알림 — 긴급 조치 불필요
NORMAL 초록색 상황이 정상으로 복귀됨
연결 상태 표시:
  • 초록 점 + "Connected": 라이브 피드가 활성화되어 이벤트를 수신 중
  • 주황 점 + "Reconnecting...": 연결이 끊어짐 — 15초 후 자동 재연결
최대 표시 이벤트: 최신 15개 이벤트만 표시됩니다. 새 이벤트가 들어오면 오래된 이벤트는 자동으로 제거됩니다.
9

생산 최적화 제안

최근 14일간의 OEE 데이터를 분석하여 개선 우선순위를 카드 형태로 표시합니다. 어느 라인에 집중해야 가장 큰 OEE 향상을 얻을 수 있는지 알 수 있습니다.

우선순위 기준

P1 — 긴급 (Critical)

OEE 목표 대비 20%p 이상 미달. 즉각적인 조치 필요.

P2 — 높음 (High)

OEE 목표 대비 10~19%p 미달. 이번 주 내 조치 권장.

P3 — 보통 (Medium)

OEE 목표 대비 10%p 미만 미달. 지속적 개선 권장.

최적화 카드 읽는 법

카드 요소 설명
우선순위 배지 (P1/P2/P3) 긴급도 수준 — 위 기준 참고
트렌드 화살표 (↑ / → / ↓) 해당 라인 OEE가 상승 중인지, 유지 중인지, 하락 중인지
현재 OEE 해당 라인의 14일 평균 OEE
-X%p below target 85% 세계 수준 목표(85%)까지 남은 차이
Current 바 현재 OEE를 시각화한 막대. 세로 선이 85% 목표를 나타냄
Potential 바 병목 요소를 해결했을 때 예상되는 OEE
Bottleneck (병목) OEE 구성 요소 중 목표 대비 가장 큰 격차가 있는 항목
Avail. / Perf. / Quality 가용률·성능률·품질률의 14일 평균. 병목 요소는 강조 표시됨
vs. best line 공장 내 최고 성과 라인 대비 이 라인이 얼마나 낮은지
Improvement Suggestions 클릭하면 병목 유형 및 격차 크기에 맞는 2~4개 개선 제안이 펼쳐짐

병목 유형

⚙️ 가용률 (Availability)

설비가 계획된 시간보다 적게 가동되는 경우. 주요 원인: 비계획 다운타임, 늦은 시동, 교체 작업.

🏃 성능률 (Performance)

설비가 가동되지만 정격 속도보다 느리게 운전되는 경우. 주요 원인: 마이크로 정지, 속도 저하, 공구 마모.

🔬 품질률 (Quality)

불량품 또는 재작업 비율이 높은 경우. 주요 원인: 자재 문제, 공정 이탈, 작업자 오류.

"잠재 OEE(Potential OEE)" 계산 방법: AI가 병목 요소를 목표 수준까지 개선했을 때 OEE가 얼마나 향상될지 추정합니다. 예상 향상량은 잠재 바에 +X%로 표시됩니다.

용어 정의

OEE (종합 설비 효율, Overall Equipment Effectiveness)
생산성을 측정하는 지표. OEE = 가용률 × 성능률 × 품질률. 세계 수준 목표: 85% 이상.
가용률 (Availability)
계획된 생산 시간 중 설비가 실제로 가동된 비율. 목표: 90%.
가용률 = 가동 시간 / 계획 생산 시간
성능률 (Performance)
설비의 이론적 최대 속도 대비 실제 가동 속도의 비율. 목표: 90%.
성능률 = 실제 생산량 / 이론적 최대 생산량
품질률 (Quality)
전체 생산 수량 중 불량 없이 생산된 양품의 비율. 목표: 99%.
품질률 = 양품 수량 / 전체 생산 수량
Z-Score
어떤 값이 평균에서 몇 표준편차 떨어져 있는지를 나타내는 통계 지표. Z-Score 2.0 이상은 통계적으로 유의미하게 낮은 OEE 값을 의미합니다.
기준선 (Baseline, 30일)
이상 감지의 기준이 되는 특정 라인의 최근 30일 OEE 평균 및 표준편차.
위험 점수 (Risk Score)
다운타임 빈도, 다운타임 비율, 마지막 정비 이후 경과 시간을 기반으로 계산된 고장 위험 추정치 (0~100점).
건강지수 (Health Index)
7일 OEE 평균을 기반으로 생산 라인의 전체 상태를 0~100점으로 요약. 높을수록 좋음.
병목 (Bottleneck)
OEE 구성 요소(가용률·성능률·품질률) 중 목표 대비 가장 큰 격차가 있어 전체 OEE를 가장 많이 제한하는 항목.
SSE (서버-전송 이벤트, Server-Sent Events)
서버가 지속적인 HTTP 연결을 통해 브라우저에 실시간 업데이트를 전송하는 기술. 실시간 스트리밍 피드에 사용됩니다.
신뢰구간 90% (Confidence Interval)
AI가 90% 확률로 실제 OEE 값이 이 범위 안에 들어올 것으로 예측하는 구간. 범위가 넓을수록 예측 불확실성이 큽니다.
P1 / P2 / P3
생산 최적화 기회의 우선순위 분류. P1(긴급, 격차 20%p 이상), P2(높음, 10~19%p), P3(보통, 10%p 미만).

자주 묻는 질문

대시보드 일부 값이 "--"로 표시됩니다. 왜 그런가요?

AI 모듈이 데이터를 로드 중입니다. 보통 페이지 로드 후 1~3초 내에 값이 채워집니다. 10초가 지나도 값이 없다면 Refresh 버튼을 클릭해보세요.

스트리밍 피드에 "Reconnecting..."이 표시됩니다. 정상인가요?

서버와의 실시간 연결이 일시적으로 끊어진 상태입니다. 15초 이내에 자동으로 재연결됩니다. 계속 연결이 안 된다면 페이지를 새로 고침하세요.

생산 최적화 섹션에 카드가 없습니다. 왜 그런가요?

현재 모든 생산 라인이 OEE 목표(85%)를 달성하고 있다는 의미입니다. 개선 카드는 목표 미달 라인에만 표시됩니다.

대시보드를 얼마나 자주 확인해야 하나요?

대시보드는 자동으로 계속 갱신되므로 모니터에 항상 띄워 두셔도 됩니다. 선제적 관리를 위해 교대 시작 시 예방정비 권고와 생산 최적화 섹션을 점검하시기 바랍니다.

AI 개선 제안은 항상 정확한가요?

AI 제안은 통계적 패턴과 업계 모범 사례를 기반으로 합니다. 조사의 출발점으로 활용하시고, 실제 조치 전에는 반드시 현장 점검 및 작업자 경험과 함께 판단하시기 바랍니다.

날짜 범위를 "Last 30 Days"로 선택하면 데이터가 느리게 로드됩니다. 정상인가요?

장기 기간(30일)은 분석 데이터 양이 많아 로딩 시간이 다소 길 수 있습니다. 실시간 모니터링에는 Today를 사용하고, 장기 분석이 필요할 때만 30일을 선택하시는 것을 권장합니다.

High-Risk Machines 값과 배지 숫자가 다를 수 있나요?

카드 값은 DANGER + CAUTION 합계이며, 배지는 수준별로 별도 표시됩니다. 예: DANGER 0개·CAUTION 1개일 때 카드 값=1, 배지="1 CAUTION". 각 수준의 개수는 카드 서브 텍스트("N danger · M caution")에서 확인할 수 있습니다.

최적화 카드의 "vs. best line"은 무슨 의미인가요?

같은 공장 내 최고 성과 라인 대비 해당 라인의 OEE가 몇 %p 낮은지를 나타냅니다. 자사 공장에서 이미 달성하고 있는 수준을 기준으로 현실적인 목표를 설정하는 데 도움이 됩니다.
