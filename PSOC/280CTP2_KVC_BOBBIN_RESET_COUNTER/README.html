280CTP2_KVC_BOBBIN_RESET_COUNTER

최초 작성: 2026-03-05 분석 버전: V1 MCU: PSoC4 Cortex-M0 개발 중

1 프로젝트 개요

Cypress PSoC 4 기반의 보빈 리셋 카운터(Bobbin Reset Counter) 펌웨어입니다. 봉제 기계의 보빈(실패) 교체 주기를 카운팅하고, 설정된 트림 횟수 도달 시 부저 알림과 외부 신호를 출력하는 생산 관리 시스템입니다.

주요 기능

기능설명
트림 카운터외부 TC_INT 신호를 ISR로 감지하여 카운트 자동 증가
목표값 설정SET 메뉴 → Trim Count 터치로 목표 트림 횟수 설정 (+1/+10)
알림 출력목표 도달 시 부저 경보 + RESERVED_OUT_1 외부 신호 HIGH
카운터 리셋물리 RESET 버튼 또는 터치 RESET 버튼으로 초기화
데이터 저장내부 EEPROM — 전원 차단 후 카운트 복원
LCD UIST7789V 320×240 터치 디스플레이 (대형 숫자 폰트)
WiFi 통신ESP 기반 WiFi 모듈 연동 (현재 미활성화)
USB 디버그USB CDC UART 디버그 출력

4.2 트림 카운터 동작 흐름

TC_INT 핀 상승 엣지 (봉제기 트림 신호)
    └── CY_ISR(Trim_Interrupt_Routine)
            ├── g_updateTrimCount = TRUE   ← volatile 미선언 이슈!
            └── g_ptrCount->count++        ← volatile 미선언 이슈!

SetCountLoop() [메인 루프 폴링]
    ├── [RESET 조건] RESET_KEY_Read() == FALSE && count != 0
    │       ├── CyDelay(50)               ← busy-wait 이슈!
    │       ├── g_ptrCount->count = 0
    │       ├── SaveInternalFlash()
    │       ├── RESERVED_OUT_1_Write(0)
    │       └── Buzzer(BUZZER_STOP, 0)
    ├── [알림 조건] count == setTrimCount
    │       ├── Buzzer(BUZZER_WARNING_CONTINUOUS, 0)
    │       ├── RESERVED_OUT_1_Write(1)
    │       └── g_bTrimCountComplete = TRUE
    └── [저장 조건] g_updateTrimCount == TRUE
            ├── SaveInternalFlash()
            └── g_updateCountMenu = TRUE

4.4 데이터 저장 구조

내부 EEPROM (Em_EEPROM, 128 bytes)
└── INTERNAL_CONFIG
    ├── watermark   [2 bytes]  : 12345 (검증 코드 비활성화 이슈!)
    ├── data        [124 bytes]: COUNT.count (현재 트림 카운트)
    └── checksum    [2 bytes]

외부 SPI Flash (W25Qxx, 마지막 섹터 4KB)
└── EXTERNAL_CONFIG
    ├── watermark   [20 bytes] : "SUNTECH IOT"
    ├── data        [200 bytes]: 서버/화면 설정
    ├── userData    [300 bytes]: MACHINE_PARAMETER (setTrimCount)
    └── CRC         [2 bytes]  (i=1 시작 버그 이슈!)

5 프로젝트 특화 분석

5.1 조건부 컴파일 구조 (package.h)

매크로상태활성화 코드 범위

USER_PROJECT_PATTERN_SEWING_MACHINE 비활성 재봉기 패턴 머신 복잡 카운팅 구조체 + ISR
USE_WIFI 미정의 DrawWifi(), WiFi 아이콘 표시 제어

5.2 IOUtil 프레임워크 현황

lib/IODefine.h / IOUtil.h — 입출력 엣지 검출 및 타이머 추상화 프레임워크 제공.
현재 IODefine.c의 getSensor(), defineIO(), doOutput() 내 모든 실제 핀 접근 코드가 주석 처리된 상태. IOUtil 구조체(SIOUtil, SIOTUtil)도 실질적으로 미사용.

5.3 SRAM 주의사항

⚠ main.c:21 주석: "image.h를 사용하면 SRAM의 50%를 차지한다"
현재 image.h / fonts.h include가 주석 처리되어 있음. 활성화 시 심각한 SRAM 부족 발생.

6 안돈(Andon) 시스템 연동

andonMenu.c, andonApi.h 등 안돈 시스템 연동 파일이 포함되어 있으나, 현재 USER_PROJECT_TRIM_COUNT 모드에서는 안돈 API 직접 호출이 없음. 향후 서버 전송 기능 활성화 시 연동 예정으로 보임.

9 테스트 시나리오

9.1 기본 카운터 테스트

시나리오절차기대 결과
카운트 증가TC_INT 핀에 상승 엣지 신호 인가LCD 현재 카운트 +1 표시
목표 도달 알림카운트가 setTrimCount에 도달부저 경보 + RESERVED_OUT_1 HIGH
RESET 버튼RESET_KEY LOW 인가 (count > 0)카운트 0으로 초기화, EEPROM 저장
전원 차단/복원임의 카운트 상태에서 전원 OFF/ONEEPROM에서 카운트 복원

9.2 목표값 설정 테스트

시나리오절차기대 결과
